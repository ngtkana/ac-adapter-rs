pub fn fft<const P: u64>(items: &mut [Fp<P>])
Expand description

Computes the Fast Fourier Transform of a field element array (in-place).

Applies the Cooley-Tukey FFT algorithm to compute the Discrete Fourier Transform.

For an input array $(x_0, x_1, \ldots, x_{n-1})$, computes: $$X_{\mathrm{rev}(i)} = \sum_{j=0}^{n-1} x_j \cdot w_n^{i \cdot j}$$ where $w_n$ is a primitive $n$-th root of unity over $\mathbb{F}_P$.

The implementation uses the in-place Cooley-Tukey algorithm with bit-reversal permutation. The transform is performed in-place, modifying the input array directly. This is typically followed by pointwise operations and an inverse FFT to compute the result in the coefficient domain.

§Preconditions

  • Array length must be a power of two
  • Array length must not exceed $2^k$ where $k$ is the largest power of 2 dividing $P - 1$

§Time Complexity

$O(n \log n)$ where $n = $ items.len()

§Examples

use fp_fft::fft;
use fp::fp;

const P: u64 = 998_244_353;
let mut data = [fp::<P>(3), fp::<P>(5)];
fft(&mut data);
// FFT of [x, y] is [x + y, x - y]
assert_eq!(data[0], fp::<P>(8));         // 3 + 5
assert_eq!(data[1], fp::<P>(998244351)); // 3 - 5 ≡ -2 (mod P)
